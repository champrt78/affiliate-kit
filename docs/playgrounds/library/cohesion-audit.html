Component library — cross-component cohesion audit

OLD (current, left) versus NEW (harmonised, right) for each drifted design primitive across the 12 mockups. Per-site colour and font are intentional and untouched. This pass only harmonises the shared structure: corner radius, button shape and text size, the section eyebrow treatment, the award label, card shadow, and two stray token values. Each row renders the real component CSS so you can compare directly.

Honest headline: the library is already largely one product. Every component lives in its own file with a single structural definition skinned by the same five token blocks, on the same scaffold. The drift is all in polish primitives, not in layout or identity. Seven findings below, ranked by how much each one hurts the "one product" feel. Fixing the top three (radius, buttons, the Bottom Line eyebrow) closes most of the gap.

Rank 1 · highest impact
Card corner radius is three different values
at-a-glance card 6px · deep card 12px · Bottom Line, guide and review cards 14px

What changes: every content card converges on 14px, the value three of the five shipping cards already use. The at-a-glance card jumps from a much tighter 6px (the lone outlier that reads as a different product), and the deep card nudges up from 12.

Why: radius is one of the first things the eye reads as "same family or not." A 6px card next to a 14px card looks like two different sites. Inner white product boxes get their own canonical 10px so the image corner relates to the card corner without matching it.

Affected: 05 at-a-glance (6 → 14), 06 deep card (12 → 14), 04 Bottom Line card, 07 guide and review cards (already 14, kept). Image boxes in 04, 05, 06, 11 normalise to 10px.

Rank 2 · highest impact
Buy buttons drift on radius, text size and shadow
radius 2 / 6 / 8 / 9 px · text 11 / 12 / 13 / 13.5 / 15 px · some flat, some shadowed

What changes: one button shape everywhere. Radius 8px, text 14px / 700, padding 13px by 20px, and a single resting shadow. The Quick Picks button (2px, near-square) and the deep-card rail button (6px, uppercase 13px) were the furthest off.

Why: the buy button is the single most-repeated interactive element and the one users click. It should be muscle-memory identical on every page. Today the same action looks like five slightly different controls. Button text size drifting from 11 to 15px is as visible as the radius.

Affected: 03 hero, 04 Bottom Line (9 → 8, 15 → 14), 05 at-a-glance (13.5 → 14), 06 deep rail (6 → 8, uppercase dropped), 08 comparison (11 → 14), 10 CTA (canonical reference, kept), 11 Quick Picks (2 → 8, 12 → 14).

Rank 3 · high impact
The Bottom Line eyebrow uses a different recipe than every other section
Bottom Line: serif + brass + 22px rule · FAQ: rule-less centred · hero and section-header: sans + accent + 26px rule

What changes: all section-level eyebrows use the recipe the hero and section-header already settled on: sans · 12px · .16em · accent colour · a 26 by 2px leading rule. The Bottom Line eyebrow drops its serif face and brass colour. The FAQ eyebrow gains the leading rule instead of floating centred.

Why: the eyebrow is a recurring signpost. When the same semantic element (a small label above a block) is serif-brass in one place and sans-accent in another, the page loses its rhythm. Card category labels stay a separate, smaller tier-2 treatment on purpose, so this is not "make every label identical," only the section-level ones.

Affected: 04 Bottom Line (serif → sans, brass → accent, 22 → 26 rule), 09 FAQ (add rule, left-align with block). Already canonical: 03 hero, 12 section-header. Untouched: 07 listing kicker and 11 Quick Picks category (tier-2 micro-kicker).

Rank 4 · medium impact
Award labels are a banner in one card and a pill in another
at-a-glance card: full-width top banner · deep card: rounded inline pill

What changes: the rounded pill becomes the canonical award shape. It works inline at the top of a body and as a corner tag, and it matches the price-badge pill already in use. The full-width banner is kept only as a deliberate variant for the at-a-glance hero card, not as a second default.

Why: "Best Overall / Best Value / Best Budget" is the same label doing the same job in both card tiers. Two different physical shapes for one semantic award make the two card types feel authored separately. Picking one shape lets the award read as a system element.

Affected: 05 at-a-glance (banner → pill, banner retained as opt-in hero variant), 06 deep card (pill, kept). Price badge in 05 already a pill, reinforcing the choice.

Rank 5 · medium impact
Cards are flat-with-border except the Bottom Line card, which floats
at-a-glance, deep, guide, review: no shadow · Bottom Line card: 0 6px 22px

What changes: one canonical resting shadow on all content cards: 0 2px 10px rgba(0,0,0,.06). The Bottom Line card steps down from its deeper 0 6px 22px so it sits in the same plane as everything else. Buttons keep their own slightly stronger shadow, giving the system three shadows total: card rest, button rest, button hover.

Why: elevation is a depth signal. When most cards are flat and one card visibly floats, that one card reads as more important than it is, and the page depth feels uneven. A single quiet card shadow makes every card sit at the same height.

Affected: 04 Bottom Line card (deeper → canonical), 05 at-a-glance, 06 deep card, 07 guide and review cards (flat → canonical rest shadow). 01 frame shadow is gallery chrome and does not ship.

Rank 6 · token bug
FussyBean accent colour disagrees between files
navbar, footer, hero use #d59b54 · the other nine files use #b06f2c

What changes: pick one FussyBean accent. The plurality is #b06f2c (the caramel that also doubles as the brass token in those files), used in nine of twelve. The three early files (navbar, footer, hero) and one file with no accent at all (CTA) get reconciled to it.

Why: the accent drives the buy button, the eyebrow, and link hovers. Two caramels across the same site means the call-to-action colour shifts page to page. This is a copy-paste skew from the files being built by separate agents, not a design intent. One value, applied everywhere.

Affected: 01 navbar, 02 footer, 03 hero (#d59b54 → #b06f2c), 10 CTA (no accent defined → add #b06f2c). The lighter #d59b54 can stay as a brass-soft style highlight if wanted, but not as --accent.

Rank 7 · token bug
GameOverGear brass-soft is overridden in one file
eleven files: #2c2752 (deep indigo) · listing-cards: #9a93c7 (lavender)

What changes: restore the canonical gog --brass-soft: #2c2752 in the listing-cards file. The lavender #9a93c7 there was a local patch so the dark guide card's kicker stayed legible, but it silently redefines a shared token.

Why: brass-soft is used for hairline accents and the FAQ icon background. A one-file override means the same token paints two different colours depending on which component you land on. Fix the legibility need at the component level (set the kicker colour directly on the dark guide card) rather than by bending the shared token.

Affected: 07 listing-cards (#9a93c7 → #2c2752, with the guide-card kicker recoloured locally so it stays readable on the dark card).
