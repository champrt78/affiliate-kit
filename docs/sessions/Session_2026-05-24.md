## Decisions

- **System.Drawing.Image for dim parsing, not hand-rolled JPEG.** First attempt at parsing JPEG dimensions in PowerShell failed three times — first version picked the first SOF marker (caught EXIF thumbnails), second picked largest SOF but only fetched 64KB so the real SOF wasn't even in range, third had a parser walk-bug where after a SOF it advanced by 2 bytes instead of by segment length. Ended up using `Add-Type -AssemblyName System.Drawing` and `[System.Drawing.Image]::FromStream($memStream)` — gives canonical width/height in one call. Lesson: don't roll a JPEG parser when .NET ships one in-box.
- **Aspect tolerance 0.35..2.5, not 0.5..2.0.** Real tall product bottles (Gyeon Q²M Bathe at 500×1143 = 0.44, Adam's Wheel Cleaner at 605×1500 = 0.40) are legit portrait photos and would false-fail at 0.5 min. Widening to 0.35 covers them without admitting wordmark-banners (which sit at 2.5+).
- **Skip aspect-check on Unsplash/Pexels scene CDNs.** Atmospheric photos aren't product images and don't need aspect validation. Still run HTTP + content-type + min-size checks on them.
- **`-LiteralPath` everywhere for Astro routes.** `Get-Content -Path` interprets `[...slug].astro` as a wildcard pattern; `-LiteralPath` treats brackets literally. Same fix needed in both lint scripts.

## Discoveries / Gotchas

- **Amazon `_AC_SL1500_` JPEGs carry EXIF thumbnail SOFs in the first ~5KB.** A naïve JPEG dim-parser that picks the first SOF will report the thumbnail's dimensions (typically 244×119 or similar 2:1ish aspect) instead of the real image's. Three sequential attempts to outsmart this in pure PowerShell all failed before falling back to System.Drawing.
- **Amazon CDN size suffixes matter and the original URLs had been stripped.** The 5 broken images were all the same Amazon image ID as the new working URLs — just missing the `_AC_SL1500_` (or `_AC_SY300_SX300_QL70_FMwebp_`) size-render suffix. Without a suffix, Amazon serves an oddly-cropped default render that happens to be sliver- or banner-shaped. The `add-link.ps1` scaffolder or wherever these URLs first came from is dropping the size suffix. Worth investigating next session.
- **Image lint earned its keep on first run.** Yesterday's CE frontend audit explicitly looked for image inconsistencies and missed 5 of them. The lint script caught all 5 on its first invocation. The whole point of "build into the systems things to avoid this" — durable safeguard infrastructure that doesn't rely on me eyeballing every page.
- **`start ""` from Bash doesn't pass through to Windows shell reliably.** Switched to `Start-Process` via PowerShell tool, which always opens the URL in the default browser.

## Addendum (post-investigation)

### Root cause of broken images — staleness, not stripping (task #51 closed)

Initial hypothesis was that the scaffolder was stripping `_AC_SL1500_` size suffixes from Amazon image URLs. Investigation showed that's wrong on two fronts:

1. **`scripts/add-link.ps1`, `new-review.ps1`, `buyers-guide.ps1`, and the scaffold-piece playbook never touch image URLs.** Ray pastes them in directly from research notes; nothing in our pipeline strips or transforms them.
2. **Of the 5 "broken" URLs we just swapped, only 2 had genuinely-different image IDs (Gyeon, MTM).** The other 3 (Adam's, P&S, CarPro) used the same image IDs as the "fixed" URLs Ray pasted — my hand-rolled JPEG parser was producing false-positives by reading EXIF thumbnail SOFs instead of real SOFs (the bug that drove the switch to System.Drawing.Image).

The real root cause is **Canopy `mainImageUrl` represents Amazon's main image AT FETCH TIME, and Amazon's "main" pointer rotates over weeks — sometimes to brand logos, sometimes to wordmarks, sometimes to a variant thumbnail.** Static scaffold-time URLs go stale. Short-term mitigation is the lint script; long-term fix is build-time refresh from Canopy or a Worker proxy.

3 of the 5 fixes were therefore unnecessary in retrospect (parser false-positives), but the new URLs are still valid product photos — no point reverting. Net effect: 2 real fixes + 3 cosmetic refreshes.
