## Discoveries / Gotchas (added)

- **Wikimedia `Special:FilePath/<filename>?width=N`** is a stable image API that 302-redirects to the correct thumbnail on upload.wikimedia.org. Browsers follow it natively in `<img src>`. Don't try to construct the hash-based upload URL by hand — use Special:FilePath and let Wikipedia do the lookup. Useful any time we need to embed a verified public-domain image without scraping.
- The Edit tool `new_string` parameter is positional-strict — a typo (`new_style`) fails the whole call with InputValidationError; nothing partially applies. Reads as Claude-side handler issue, not a file-state issue.

## Discoveries / Gotchas (added)

- **Unsplash internal API works for image search.** `https://unsplash.com/napi/search/photos?query=...&per_page=N` returns JSON with photo IDs, URLs, and metadata — no API key needed for reads. Filter on `r['premium']` or `r['plus']` to skip Unsplash+ paid photos. Then `https://unsplash.com/napi/photos/{id}` returns full URLs. Useful for any "find a free photo" task without burning Firecrawl credits or hand-curating.
- **Unsplash photo URLs are stable and CDN-hosted at `images.unsplash.com`.** Hot-linking is the intended use; license allows commercial without attribution. Resize/crop via query params (`?w=600&h=600&fit=crop&q=85&auto=format`). No need to download and self-host for decorative images.
- **`object-fit: cover` + `object-position` is the right tool for re-cropping a photo into a badge slot.** Got us through the J4-windshield-crop iteration. Setting `object-position: 30% 0%` shifts visible area to the top-left when the photo's subject is in that corner.
- **CSS `filter: saturate(1.35) brightness(1.04) contrast(1.05)`** is a good "punch" preset — feels editorial without being garish. Heavier `saturate(1.6)` reads as magazine-ad. Don't crank past 1.6 unless the look is intentionally overcooked.
- **Iterative-mockup pattern works.** Built 5 mockups (F, G, H, I, J, K) in sequence — each one branching off Ray's feedback on the prior. Faster than guessing at the right photo+layout combo upfront. Each mockup stayed at `docs/playgrounds/quickpicks-mockups/` so we can revisit the design space if needed later.
