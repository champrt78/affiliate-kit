## Discoveries / Gotchas
- **Root cause of the recurring header split**: `.page-header__inner` carried a stale `display:grid` — the listing pages had the *original* 2-col `minmax(0,1fr) minmax(0,1fr)`; the guide-indexes had `auto-fit` grid **bled in from my earlier card-grid find-replace sweep**. Both were masked by an appended `display:block` override patch. So every header was "a grid rule fighting an override patch" — drift-prone, and the source of the "header is only half the page" complaint. Cleanup removes the grid entirely; render is unchanged because the override already forced block (verified).
- `--color-forest` is NOT bleed — it's a deliberate **shared semantic token** (heading-ink slot); each skin maps its own ink onto it (espresso #241710, teal #0c5470, gog light-ink #ede9ff). mwc references it but defines it in a base/token file, not site.css (out of scope, not flagged by audit).
- agent-browser device emulation is **macOS-only** — can't set a mobile viewport on Windows. Used the Playwright MCP (its own headless Chromium) for the 390px mobile check instead.
- auto-fit + `justify-content:center` on a lone card computes to tracks `300px 0px 0px` — centered per computed style, but collapsed-track gaps lean it slightly left. Negligible; the real bug ("card too big") is fixed by the 300px cap.
