### Discovery / gotcha — the "plugin" was never a plugin

The biggest discovery of this round: `~/.claude/plugins/affiliate-kit/` was not a Claude Code plugin in the registry sense. Claude Code's plugin loader only picks up plugins registered via marketplaces (compound-engineering, posthog, atlassian, etc.) — not arbitrary folders under `~/.claude/plugins/`. The `aff-bootstrap` and `aff-help` commands we'd been "installing" for 6 days were never invokable. The real loader is `~/.claude/commands/` for user-global slash commands; that's why the 4 new commands worked (because they live there) and the old ones didn't (because they lived in the unread plugin folder).

This matters for the consolidation design: source of truth in repo, install destination is `~/.claude/commands/`. The plugin folder at `~/.claude/plugins/affiliate-kit/` keeps only `config.json` (Cloudflare secrets) and `README.md`/`plugin.json` for documentation; no commands.
