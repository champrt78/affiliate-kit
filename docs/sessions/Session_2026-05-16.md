After the Phase B walkthrough discussion, Ray pushed all the way through to actually wire GitHub auto-deploy for mywildlifecam (which CF's docs say isn't supported for retrofit). Six distinct friction points hit and fixed:

1. **CF Pages does NOT support retrofitting Git onto direct-upload projects.** Workaround: create a new Workers project from Git that publishes static output to a separate Pages project.
2. **CF "Connect to Git" creates a Workers project, not a Pages project.** Workaround: leave it as Workers, override the Deploy command to use `wrangler pages deploy` explicitly.
3. **Default Build command (`pnpm run build`) ran a recursive build that surfaced two pre-existing bugs in `tools/bootstrap`:** tsconfig rootDir/include mismatch + missing `.js` extensions on relative imports in NodeNext mode. Both fixed in `e6e6c07`.
4. **"API token" field in CF dashboard doesn't inject `CLOUDFLARE_API_TOKEN` with sufficient permissions.** Workaround: create a token explicitly with `Cloudflare Pages: Edit` permission (Workers-template tokens DON'T include this — Pages is a separate permission) and either set it via the field or as a `CLOUDFLARE_API_TOKEN` secret.
5. **`wrangler pages deploy` doesn't auto-create the target Pages project.** Run `npx wrangler pages project create <slug> --production-branch=main` once before first deploy. The interactive prompt may not honor the `--production-branch=main` flag — type `main` at the prompt.

## Late addendum #9 — Caught: CF Pages was never wired to GitHub auto-deploy

Discovered during the post-piece-#3 live-URL verification (option C from earlier). All 3 piece URLs returned HTTP 200 but the HTML grep for the Bottom Line text returned empty matches. Investigation showed: **all 3 pages were serving the original home-page content from the 2026-05-12 bootstrap**, not the just-shipped pieces.

Root cause: the 2026-05-12 bootstrap created the 5 CF Pages projects via `wrangler pages project create` + `wrangler pages deploy` (direct upload). So pushes to `origin/main` go to GitHub but do not trigger CF Pages rebuilds. The live sites stay frozen at the last manual `wrangler pages deploy`.

This was a **silent failure mode** that would have bitten every future piece. Pieces 1-3 would have shipped to git but stayed invisible on the live site indefinitely.

**Immediate fix:** ran `npx wrangler pages deploy dist --project-name affkit-mywildlifecam --branch main` to push the 3 pieces live. Verified after deploy:

- `mywildlifecam.com/buyers-guides/best-trail-cameras-for-backyard-wildlife/` — "Don't overthink it" present (rendered with smart-quote apostrophe `'`, U+2019, per Astro markdown rendering — worth knowing if you ever grep for an apostrophe), robots: index/follow

- `mywildlifecam.com/reviews/stealth-cam-ds4k-ultimate-review/` — "Buy once, cry once" present, robots: index/follow
