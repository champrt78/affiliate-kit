## Late-session: First AIOS workflow tied into the dashboard

Ray's framing: *"Id like to see this tied into aios, but I want you to work without me and dont stop, if you run into something that needs me pivot to something else if you can."*

Built the **`affiliate_link_health`** AIOS workflow end-to-end. Lives at `C:\Users\Ray\documents\github\AIOS\agents_system\workflows\affiliate_link_health\` and follows the template's tool / validator / orchestrator pattern.

### What it does

Three phases:
1. **DISCOVER** — walks `sites/<slug>/src/content/{reviews,buyers-guides}/*.md` across all 5 affiliate sites, parses YAML frontmatter (no PyYAML dep — handwritten minimal parser), extracts `product.affiliate.amazon`, `products[].affiliateUrl`, `images.hero`, `images.heroImages[]`, plus constructs the live `https://<site>.com/go/<slug>` cloaker URL for every piece.
2. **CHECK** — HEAD-requests every URL (falls back to GET stream on 405). Tags healthy/unhealthy against `[200, 301, 302, 303, 307, 308]`.
3. **REPORT** — writes markdown + JSON to `data/affiliate_link_health/<run_id>.{md,json}`. Healthy/unhealthy counts roll up into the execution's `outcome_quality` for SmallBrain analysis later.

### Files added (AIOS repo)

- `workflows/affiliate_link_health/config.py` — WORKFLOW_NAME, AFFILIATE_REPO_PATH, SITE_SLUGS, HEALTHY_STATUSES (kept as a list for JSON-serializable orchestrator logging — fix below), success metrics weighted 0.70 link_health_rate / 0.20 completion / 0.10 consistency.
- `workflows/affiliate_link_health/tools/discover_urls_tool.py` — frontmatter walker.
- `workflows/affiliate_link_health/tools/check_status_tool.py` — HEAD-request checker with HEAD→GET fallback and per-request duration tracking.
- `workflows/affiliate_link_health/tools/report_tool.py` — markdown + JSON report.
- `workflows/affiliate_link_health/validators/{urls_discovered,status_checked,report_generated}_validator.py` — 3 thin validators returning the orchestrator-expected `{passed, issues, needs_more, metadata}` shape.
- `workflows/affiliate_link_health/{orchestrator,brain}.py` — template-copied.
- `workflows/affiliate_link_health/run.py` — entry point that registers the workflow + creates an executions row + calls SimpleOrchestrator + updates outcome_quality on completion.

### Dashboard wiring

### Discoveries / Gotchas

- **AIOS validator contract is `{passed, issues, needs_more, metadata}`** — `issues` is a list of strings, `needs_more` controls retry. Returning `{feedback}` matches the docs but not the orchestrator's actual KeyError surface.
- **`tool_params` round-trips through JSON via ExecutionLogger.** Anything non-serializable (sets, datetimes, Path objects) will crash the orchestrator's `unexpected error` branch. Use lists, then `set()` inside the tool's execute().
- **The AIOS orchestrator catches the JSON exception generically.** The actual stack trace doesn't reach stdout — only `Error: 'issues'` or `Error: Object of type set is not JSON serializable`. Painful to diagnose first time; trivial after the fact.
- **PROD KV writes need `--remote`.** Already in `add-link.ps1`, already in the docs, but worth re-noting after every catch-it-with-AIOS session.
