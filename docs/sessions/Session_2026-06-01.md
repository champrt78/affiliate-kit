## Discoveries / Gotchas
- **curl needs `--compressed` for Amazon dp scraping (overnight SA task).** Without it, Amazon serves brotli/gzip and the raw bytes are binary garbage — grep finds no title, no hiRes, no images, and a guessed ASIN looks "valid" at HTTP 200. Adding `--compressed` decodes the page and the `<title>` + `"hiRes":` extraction works. Title-match against the page is the discriminator that caught a guessed ASIN resolving to "Tasty Bite Madras Lentils." Resolve ASINs via `/s?k=` search (grep `data-asin="[A-Z0-9]{10}"`) — also worked with `--compressed`, no captcha — never hand-guess 10-char strings.
- **Eheim Jager hero image fails the aspect lint as-is.** The only image Amazon serves for B003SNU1I2 is the slim heater tube at 162x888, aspect 0.182 — below the lint's 0.35..2.5 floor. Fix: swap the CDN size suffix from `._AC_SL1000_` to `._AC_SS1000_` (scale-square), which pads the tube onto a clean 1000x1000 canvas, aspect 1.0. `_SS` / `_SL` square transforms are the escape hatch for inherently-narrow product images that would otherwise fail the pre-commit image lint.
- **Astro `description` max is 160 chars and the build hard-fails on overflow** (`String must contain at most 160 character(s)`), not a warning. 3 of 4 SA drafts overflowed (162/170/162) on first write. Count descriptions before building.
- **Pillar-silo trap (caught before it bit):** adding `pillar:` to a `.md` without first adding the field to the Zod content schema makes Astro silently DROP the key (unknown frontmatter is stripped). The build stays green, but every hub would render "Nothing published" because `e.data.pillar` is undefined. Fix: add `pillar: z.string().optional()` to ALL THREE collections (reviews + buyers-guides + comparisons) in `content/config.ts` BEFORE tagging. Satellites only needed it on two collections; MWC/DTP need three because they have `comparisons`.
- `lint-content-frontmatter.ps1` only scans `buyers-guides|reviews` dirs (line 54 regex) — it does NOT check comparisons. So comparisons must still be tagged for the hub to render them (schema FK), but the lint won't catch an untagged comparison. The pillar check also only activates once `topics/[pillar].astro` exists for a site (which is why MWC/DTP passed with zero pillars before today).
- MWC + DTP `MainLayout.astro` did NOT import `siteConfig` (the satellites do). First MWC build failed `siteConfig is not defined` at the 404 page until I added the import. Easy to assume parity that isn't there.
- WebFetch is the WRONG tool for Amazon og:image extraction. It converts the page to markdown and runs a small model on the BODY, dropping `<head>` meta tags entirely. Returned only CSS/JS asset image URLs. Advisor flag.
- The working method for new product images: `curl -s -A "<chrome UA>" https://www.amazon.com/dp/<ASIN>` then grep for `"hiRes":"https://m.media-amazon.com/images/I/..."`. The /dp/ page is far less bot-walled than the /s?k= search endpoint (which returned empty). First hiRes per page is the main hero image.
